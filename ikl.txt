Commands to demonstrate creating a project in local and remote repository using Git Bash and GitHub, applying `init`, `status`, `log`, `add`, `commit`, `push`, `config`, `clone`, and `reset` in Git`.
First time pushing code into a new repository
Step:1: git config --global user.name "YourName"
	git config --global user.email "your_email@example.com"
Step 2: git init
Step 3: git status
Step 4: git add .
Step 5: git commit -m "Initial commit"
Step 6: git log
Step 7: git remote add origin https://github.com/username/repository-name.git
Step 8: git push -u origin master
Step 9: git clone https://github.com/username/repository-name.git
Step 10: git reset --soft HEAD~1
	    git reset --hard HEAD~1

If the repository already Exits
Step 1: git config --global user.name "YourName"
	git config --global user.email your_email@example.com
Step 2: git clone https://github.com/username/repository-name.git
Step 3: cd repository-name
Step 4: git status
Step 5: git add .
Step 6: git commit -m "Updated commit"
Step 7: git log
Step 8: git push -u origin master
Step 9: git reset --soft HEAD~1
Step 10: git reset --hard HEAD~1

A-2
Demonstrate how to create a project in a remote repository and apply fork, merge, diff, merge conflict, branch, and pull request concepts on the repository using GitHub.
	Step1: Create Repository in GitHub
		Open GitHub.
		Click New Repository.
		Enter repository name: git-demo

	Step 2: Clone Repository to Local System
		Open terminal and run:
		git clone https://github.com/username/git-demo.git
		cd git-demo

	Step 3: Create a file and Write the content 
		Hello GitHub

	Step 4: Commit and Push
		git add file1.txt
		git commit -m "Initial commit"
		git push origin main

	Step 5: Create and Switch to Branch
		git branch feature1
		git checkout feature1
	
	Step 6: Modify File in Feature Branch
		Open file1.txt and add a line
		Now Commit changes 
		git add file1.txt
		git commit -m "Feature branch update"
		git push origin feature1

	Step 7: Show Difference Between Branches
		git diff main feature1
		
	Step 8: Create Merge Conflict
		git checkout main
		Open file1.txt and add another line
		git add file1.txt
		git commit -m "Main branch change"
		git merge feature1

	Step 9: Resolve Conflict
		Open file1.txt and add another line
		git add file1.txt
		git commit -m "Merge conflict resolved"
		git push origin main

	Step 10: Fork and Create Pull Request
		Open repository in GitHub.
		Click Fork.
		Modify the file in forked repository.
		Click Pull Request.
		Click Create Pull Request to request merge into original repository.
		

A-3   
Demonstrate the process of integrating a GitHub repository with Jenkins to automate project execution in a CI/CD pipeline.
Requisites: GitHub Repository with Java/Python files
	           Jenkins(Go to https://www.jenkins.io, Go to https://www.jenkins.io)

To Unlock Jenkins:
	        Open a browser and go to: http://localhost:8080
	        Jenkins will ask for the Administrator password.
	        Copy the password from: C:\Program data\Jenkins\secrets\initialAdminPassword
	          Install Suggested Plugins
	        
Manual Build creation
	In Jenkins:
		Click New Item
		Enter project name
	              Select Freestyle Project
		Click OK

            Configure GitHub Repository
		In Source Code Management
			Select Git
			Enter repository URL
		Add Build Step
			Scroll to Build → Add Build Step
			Execute Windows Batch Command
			Give the command (echo, python, java)
			Save the Job
			Run Build Now

Scheduled Build or Periodic Build (using Poll SCM)
	In Jenkins:
		Click New Item
		Enter project name
	              Select Freestyle Project
		Click OK
	Configure GitHub Repository
		In Source Code Management
			Select Git
			Enter repository URL
		Enable Scheduled Build 
			Scroll to Build Trigger
			Select Poll SCM
			Mention * * * * *
			Scroll to Build
			Add Build Step → Execute Windows Batch Command
			Give the command (echo, python, java)
			Save
Make a change in the GitHub repository and the code.
Jenkins will detect the change using Poll SCM and automatically start the build.		
Event-Based Automatic Build using GitHub Webhooks
		In Jenkins:
			Click New Item
			Enter project name
	              	Select Freestyle Project
			Click OK
		Configure GitHub Repository
			Scroll to Source Code Management
			Select Git
			Enter the GitHub repository URL
			Scroll to Build Triggers
			Select: GitHub hook trigger for GITScm polling
		Install and Run ngrok
			ngrok http 8080 (Jenkins port)
		

		Add Build Step
			Scroll to Build → Add Build Step
			Execute Windows Batch Command
			Give the command (echo, python, java)
			Save the Job
		Configure Webhook in GitHub
			Open your repository in GitHub
			Go to Settings
			Click Webhooks
			Click Add Webhook
			Enter Webhook Details:
			Payload URL: http://ngrok_generated_IP/github-webhook/
			Click Add Webhook
			Make a change in the project and push the code
When code is pushed to GitHub, the Webhook sends a request through ngrok to Jenkins, and Jenkins immediately starts the build.8899hhjjkkhello
